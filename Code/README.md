## 1) Bisection Method (`bisection.py`)

### แนวคิด
เป็นวิธีแบบ **Bracketing** (ต้องมีช่วงเริ่มต้น `[a,b]` ที่คร่อมราก)

เงื่อนไขสำคัญ:
- ต้องมี `f(a)*f(b) < 0`
- ฟังก์ชันต้องต่อเนื่องในช่วงนั้น

### สูตร
- จุดกึ่งกลาง  
  c = (a+b)/2
- เลือกช่วงใหม่โดยดูเครื่องหมายของ `f(a)*f(c)`

### จุดเด่น
- ปลอดภัย (ลู่เข้าแน่นอนถ้าคร่อมรากจริง)
- แต่ค่อนข้างช้าเมื่อเทียบกับ Newton/Secant

## 2) False Position / Regula Falsi (`false_position.py`)

### แนวคิด
เป็นวิธีแบบ **Bracketing** เช่นกัน แต่แทนที่จะใช้จุดกึ่งกลาง  
จะใช้ “เส้นตรงเชื่อม” ระหว่าง `(a,f(a))` และ `(b,f(b))` แล้วหาจุดตัดแกน x

เงื่อนไขสำคัญ:
- ต้องมี `f(a)*f(b) < 0`

### สูตร
xr = (a*f(b) - b*f(a)) / (f(b)-f(a))

### จุดเด่น
- เร็วกว่าบางกรณีเมื่อเทียบกับ Bisection
- แต่บางสมการอาจช้า (ค่าใกล้ด้านเดียวซ้ำ ๆ)

---

## 3) Secant Method (`secant.py`)

### แนวคิด
เป็นวิธีแบบ **Open method** (ไม่ต้องคร่อมราก)  
ใช้ค่าเดิม 2 จุด `x0, x1` เพื่อประมาณความชันแทนอนุพันธ์

### สูตร
x_{k+1} = x_k - f(x_k)*(x_k-x_{k-1})/(f(x_k)-f(x_{k-1}))

### จุดเด่น
- ไม่ต้องใช้อนุพันธ์ `f'(x)`
- มักเร็วกว่า Bisection/False Position
- แต่ต้องเลือกค่าเริ่มต้นดี ๆ

---

## 4) Newton-Raphson Method (`newton_raphson.py`)

### แนวคิด
เป็นวิธีแบบ **Open method** และต้องมีอนุพันธ์ `f'(x)`  
ใช้เส้นสัมผัส (tangent line) ที่จุด `x_k` เพื่อหา `x_{k+1}`

### สูตร
x_{k+1} = x_k - f(x_k)/f'(x_k)

### จุดเด่น
- เร็วมากเมื่อเดาค่าเริ่มต้นใกล้ราก (converge เร็ว)
- แต่ถ้าเดาไม่ดีอาจไม่ลู่เข้า หรือเจอ `f'(x)=0`

## วิธีเปลี่ยนสมการ

แก้ที่ฟังก์ชัน `f(x)` ในแต่ละไฟล์ เช่น

```python
def f(x):
    return x**3 - 4*x - 9
```

สำหรับ Newton-Raphson ต้องแก้ `df(x)` ด้วย:

```python
def df(x):
    return 3*x**2 - 2
```

## 5) Cramer's Rule (`cramers_rule.py`)

### แนวคิด
ใช้ดีเทอร์มิแนนต์ (determinant) เพื่อแก้ระบบสมการเชิงเส้น **Ax = b** โดยแก้ทีละตัวแปร

เงื่อนไขสำคัญ:
- ต้องเป็นระบบ **สมการเชิงเส้นแบบสี่เหลี่ยมจัตุรัส** (n สมการ n ตัวแปร)
- ต้องมีคำตอบเอกลักษณ์: **det(A) ≠ 0**

### สูตร
ให้ Δ = det(A)  
และ Δᵢ = det(Aᵢ) (เมทริกซ์ที่แทนคอลัมน์ที่ i ด้วยเวกเตอร์ b)

แล้ว
xᵢ = Δᵢ / Δ

### จุดเด่น/ข้อจำกัด
- เหมาะกับระบบเล็ก ๆ (เช่น 2x2, 3x3)
- ระบบใหญ่จะช้า และไม่ค่อยนิยมเมื่อเทียบกับ Gaussian elimination / LU
